Упрощенное пояснение функционала
================================

Тут будет описание всего этого прикола, что он делает и как, тут более простое описание, вся более подробная инфа написана в самом коде/документации.

Если вы видите тут звуковой файл, значит я разобрался с этим), если нет, то соболезную)

Порядок выполнения
------------------

.. dropdown:: 1. :blue:`myallpath.py`

    В данном файле используется функция :red:`select_files()` для выбора файлов:

    1. **people** - Файл с сотрудниками (.txt)
    2. **zip_path** - ZIP файл Стахановца (.zip)
    3. **zvonki** - Файл с АТС (.xlsx)
    4. **output_excel_path** - Папка для сохранения

    .. important:: Проще говоря, тут идет выбор файлов для дальнейшей работы.

.. dropdown:: 2. :blue:`sotrudniki.py`

    В данном файле используется функция :red:`parse_departments()` для создания массива с отделами и сотрудниками принадлежащим к ним, выполняется с параметром **people**.

    К примеру наш файл с сотрудниками будет выглядеть данным образом:

    .. dropdown:: Сотрудники.txt

        Отдел продаж (ОП):

        Фомина Ксения

        Щибров Артем

        Троенко Владислав

        Шакирова Екатерина

        Желтова Анастасия

        Отдел продаж 2 (ОП2):

        Тюнина Ольга

        Жесткова Юлия

        Мантрова Екатериина

        Васильева Анастасия

        Колмогорцева Виктория

    Тогда массив **mas_sotrudniki** будет

    .. dropdown:: :yellow:`mas_sotrudniki`

        mas_sotrudniki: {'Отдел продаж (ОП)': ['Фомина Ксения', 'Щибров Артем', 'Троенко Владислав', 'Шакирова Екатерина', 'Желтова Анастасия'], 'Отдел продаж 2 (ОП2)': ['Тюнина Ольга', 'Жесткова Юлия', 'Мантрова Екатериина', 'Васильева Анастасия', 'Колмогорцева Виктория']}

    .. important:: Проще говоря, тут идет обработка ранее выбранного файла с сотрудниками и создается массив.

.. dropdown:: 3. :blue:`excel_create.py`

    В данном файле используется функция :red:`create_department_employee_excel()` с параметрами **mas_sotrudniki** и **output_excel_path**.

    Создается .xlsx файл **Отчеты_отделов_и_сотрудников.xlsx** с листами по каждому отделу и сотруднику. В лист с названием отдела записываются все сотрудники, принадлежащие к нему.

    Путь до файла **Отчеты_отделов_и_сотрудников.xlsx** хранится в переменной **output_xlsx_path**.

    .. important:: Проще говоря, тут идет создание файла .xlsx из данных массива о сотрудниках с соответствующими листами.

.. dropdown:: 4. :blue:`zip_file.py`

    В данном файле используется функция :red:`unzip_file` с параметром **zip_path** в которой хранится путь до .zip файла Стахановца.

    Функция распаковывает .zip файл в папку с таким же именем.

    Путь до распакованной папки хранится в **start_path**.

    .. important:: Проще говоря, тут идет распаковка ранее выбранного файла .zip Стахановца.

.. dropdown:: 5. :blue:`clearPath.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`delete_png_files()`

        Данная функция удаляет все .png/.csv/.css/.html/.dat/.js/.htm файлы в папке распакованного архива.

        Используется с параметром **start_path**.

    .. dropdown:: 2. :red:`delete_small_folders()`

        Данная функция удаляет все папки (кроме корневой), в которых менее 5ти файлов. Так как они не несут в себе какого-либо смысла или попросту не нужны для работы приложения.

        Используется с параметрами **start_path** и **минимальным кол-вом фалов в папке**.

    .. dropdown:: 3. :red:`convert_xls_to_xlsx()`

        Данная функция превращает все оставшиеся .xls файлы в .xlsx для более удобной работы с ними.

        Используется с параметром **start_path**.

    .. dropdown:: 4. :red:`delete_x_files()`

        Данная функция удаляет все оставшиеся .xls файлы.

        Используется с параметром **start_path**.

    .. dropdown:: 5. :red:`delete_folders_based_on_C2_recursive()`

        Данная функция удаляет все папки, где в .xls/.xlsx файлах в C2 имеется слово "Телефон", так как эти файлы нам не нужны.

        Используется с параметрами **start_path** и **word='телефон'**.

    .. dropdown:: 6. :red:`delete_pacs()`

        Данная функция удаляет все .xlsx файлы у которых в названии есть pacs, так как данные файлы не нужны нам.

        Используется с параметром **start_path**.

    Данные функции предназначены для отчистки распакованного архива Стахановца от ненужных файлов.

    .. important:: Проще говоря, тут идет отчистка распакованного архива Стахановца от всякой шелухи.

.. dropdown:: 6. :blue:`search_file.py`

    В данном файле используется функция :red:`find_xml_files()` с параметром **start_path**.

    Функция ищет .xml (index.xml) файл и сохраняет его по пути **found_paths**

    .. important:: Проще говоря, тут идет поиск index.xml в распакованном архиве.

.. dropdown:: 7. :blue:`siteNprog_normolize.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`extract_report_data_prog()`

        Данная функция извлекает информацию из **index.xml** о программах с параметрами пользователя их именами, удаляет не нужные данные и записывает это в **prog_index.xml**, файл сохраняется в том же месте, где был найден **index.xml**.

        Используется с параметром **found_paths**.

        Путь до **prog_index.xml** сохраняется в **output_file_path_prog**.

    .. dropdown:: 2. :red:`extract_report_data_site()`

        Данная функция извлекает информацию из **index.xml** о сайтах с параметрами пользователя их именами, удаляет не нужные данные и записывает это в **site_index.xml**, файл сохраняется в том же месте, где был найден **index.xml**.

        Используется с параметром **found_paths**.

        Путь до **site_index.xml** сохраняется в **output_file_path_site**.

    Данные функции предназначены для обработки **index.xml** и разбора информации внутри него на 2 файла **prog_index.xml** и **site_index.xml**.

    .. important:: Проще говоря, тут идет обработка index.xml и разделение на 2 файла с сайтами и программами.

.. dropdown:: 8. :blue:`siteNprog_toexcel.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`load_program_data()`

        Данная функция выполняет загрузку данных о программах из XML файла **(prog_index.xml)** и преобразование их в DataFrame.

        Используется с параметром **output_file_path_prog**.

        Значение DataFrame сохраняется в **program_data**.

    .. dropdown:: 2. :red:`load_site_data()`

        Данная функция выполняет загрузку данных о сайтах из XML файла **(site_index.xml)** и преобразование их в DataFrame.

        Используется с параметром **output_file_path_site**.

        Значение DataFrame сохраняется в **site_data**.

    .. dropdown:: 3. :red:`update_employee_sheets()`

        Данная функция выполняет обновление листов сотрудников в Excel файле данными о программах и сайтах.

        Создаются столбцы "Дата", "Время", "Программа" "Сайт".

        Используется с параметрами **output_xlsx_path**, **program_data** и **site_data**.

        Путь до файла сохраняется в **output_xlsx_path**.

    Данные функции предназначены для вставки значений из **prog_index.xml** и **site_index.xml** в **Отчеты_отделов_и_сотрудников.xlsx** по каждому сотруднику и времени, в соответствующие листы и ячейки.

    .. important:: Проще говоря, тут идет вставка данных о программах и сайтах в .xlsx файл с листами сотрудников.

.. dropdown:: 9. :blue:`inExcel_site.py`

    В данном файле используется функция :red:`process_excel()`, функция заменяет значения в столбце "Сайты" на более читаемы, которые прописаны в :blue:`myalldata.py` в массиве :yellow:`url_mapping`.

    Файл сохраняется с именем **Отчеты_отделов_и_сотрудников_обновленный.xlsx** и путь записывается в переменную **output_file_excelSite**.

    Функция использует 2 параметра при запуске напрямую через Python и 3 параметра при запуске приложения.

    **output_xlsx_path** и **url_mapping** - постоянно используемые параметры.

    **output_file_excelSite** - параметр используемый только для приложения. Это используется для постоянного хранения пути сохранения файла и успешного переноса временных файлов в скрытую папку.

    .. important:: Проще говоря, тут идет переименовывание сайтов, что б они нормально выглядели и не занимали много места.

.. dropdown:: 10. :blue:`zvonki_normolize.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`create_file_copy()`

        Данная функция создает копию файла .xlsx со звонками из АТС, указанного пользователем

        Путь до копии фала хранится в **copy_file_path_zv**.

        Используется с параметром **zvonki**.

    .. dropdown:: 2. :red:`process_and_save_calls_data()`

        Данная функция форматирует **copy_file_path_zv** (удаляет не нужные строки/столбцы, переименовывает сотрудников, изменяет формат даты, изменяет форматы ячеек).

        Используется с параметрами **copy_file_path_zv** и **replacements**.

        :yellow:`replacements` -  массив, который прописан в :blue:`myalldata.py`.

    .. important:: Проще говоря, тут идет копирование ранее выбранного файла со звонками из АТС и его форматирование.

.. dropdown:: 11. :blue:`zvonki_toexcel.py`

    В данном файле используется функция: :red:`zvonkiExcel()`, создается словарь для хранения данных по каждому сотруднику из отдела.

    Функция содержит в себе следующие функции:

    .. dropdown:: 1. :red:`def convert_sheet_data()`

        Функция выполняет преобразование данных на листах сотрудников в подходящий формат.

    .. dropdown:: 2. :red:`insert_call_data_to_report()`

         Функция для вставки данных о звонках в соответствующие листы сотрудников.

    .. dropdown:: 3. :red:`insert_new_row_below()`

        Функция для вставки значений в листы с сотрудниками  из словаря.

    .. dropdown:: 4. :red:`insert_new_row_sorted()`

        Функция для вставки сортированных значений по дате и времени.

    Используется с параметрами **output_file_excelSite** и **copy_file_path_zv**.

    .. important:: Проще говоря, тут идет вставка данных о звонках в .xlsx файл с сотрудниками.

.. dropdown:: 12. :blue:`stahName.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`rename_folders_from_excel_cell()`

        Функция изменяет имена папок соответствуя C2 в файлах .xlsx.

        Используется с параметром **start_path**.

    .. dropdown:: 2. :red:`check_and_modify_excel()`

        Данная функция изменяет имена сотрудников, проверя значения в .xlsx файлах Стахановца в С2 на прописанные в :yellow:`replacements`.

    .. dropdown:: 3. :red:`remove_folder_if_invalid()`

        Функция для удаления папки, если она содержит файлы, не удовлетворяющие условиям, если менее 5ти файлов.

    .. dropdown:: 4. :red:`process_folders()`

        Основная функция для обхода папок и проверки файлов

        Используется с параметрами **start_path** и **replacements**.

    .. important:: Проще говоря, тут идет переименовывание файлов/папок/значений внутри .xlsx файлов Стахановца.

.. dropdown:: 13. :blue:`infoWork_stah.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`get_all_employees()`

        Функция для объединения всех сотрудников в один список

        Используется с параметром **mas_sotrudniki**.

        Сохраняется в переменную **all_employees**.

    .. dropdown:: 2. :red:`process_worktime_xlsx()`

        Функция для обработки данных из файла worktime.

        Функция записывает данные в **info_work_stah** о времени работы ПК.

    .. dropdown:: 3. :red:`process_entrexit_xlsx()`

        Функция для обработки данных из файла entrexit.

        Функция записывает данные в **info_work_stah** о входе/выходе ПК.

    .. dropdown:: 4. :red:`process_acttime_xlsx()`

        Функция для обработки данных из файла acttime.

        Функция записывает данные в **info_work_stah** об активном времени сотрудника.

    .. dropdown:: 5. :red:`scan_folders()`

        Функция сканирует все папки для поиска  .xlsx файлов содержащих в названии worktime/entrexit/acttime и запуска соответствующих функция для занесения данных в **info_work_stah**.

        Используется с параметрами **start_path** и **all_employees**.

    .. dropdown:: 6. :red:`print_pretty_info_work_stah()`

        Красивый вывод **info_work_stah**.

    .. dropdown:: 7. :red:`print_pretty_info_work_stah1()`

        Тоже красивый вывод **info_work_stah**.

    Создается массив данных **info_work_stah** в который записываются данные о времени работы пк(**worktime**), активном времени(**acttime**), входе/выходи из пк(**entrexit**), для каждого сотрудника по каждому дню.
    Данные берутся из соответствующих фалов Стахановца.

.. dropdown:: 14. :blue:`infoStah_toexcel.py`

    В данном файле используются функции:

    .. dropdown:: 1. :red:`format_time_string()`

        Функция преобразовывает формат времени в правильный('9ч 41м', '9 часов 41 минут' или '15 минут' в 'hh:mm').

    .. dropdown:: 2. :red:`update_excel_with_employee_data()`

        Функция так же работает с форматами времени/даты и вставляет значения из **info_work_stah** в .xlsx файл отчета.

        Используется с параметрами **info_work_stah** и **output_file_excelSite**.

    .. important:: Проще говоря, тут идет обработка данных в **info_work_stah** и вставка их в отчет.

.. dropdown:: 15. :blue:`bitrix_normolize.py`

    **На данный момент это не используется!**

    В данном файле используются функции:

    .. dropdown:: 1. :red:`convert_html_to_xlsx()`

        Данная функция конвертирует .xls в .xlsx.

    .. dropdown:: 2. :red:`format_time_data()`

        Функция конвертирует записи в ячейках в нужный формат.

    .. dropdown:: 3. :red:`adjust_time_for_timezone()`

        Функция преобразовывает время к правильному, отталкиваясь от часового пояса сотрудника, прописанных в **employee_timezones**.

    .. dropdown:: 4. :red:`replace_values_in_xlsx()`

        Функция преобразовывает имена сотрудников на правильные, прописанные в **bit_replacements**.

    .. important:: Проще говоря, тут идет работа с ранее выбранным файлом Битрикса и преобразование/форматирование данных внутри.

.. dropdown:: 16. :blue:`format.py`

    Данный файл содержит функцию :red:`format_excel_file()`, которая создает границы, задает ширину столбцов, создает заголовки, делает группировку данных, создает условное форматирование, делает перенос текста, удаляет столбцы и создает фильтры.

    Используется с параметром **output_file_excelSite**.

    .. important:: Проще говоря, тут идет полное форматирование уже готового отчета для читаемого/красивого вида.


.. dropdown:: 17. :blue:`dost_file.py`

    В данном файле используются функции:

     .. dropdown:: 1. :red:`fileto_log()`

        Функция "прячет" временные фалы в папку log.

        Используется с параметрами **timefiles** и **output_excel_path**.

    .. dropdown:: 2. :red:`frename_xlsx_files()`

        Функция переименовывает файлы Стахановца.

        Используется с параметром **start_path**.

    .. important:: Проще говоря, тут идет работа с временными файлами, которые остаются после выполнения.

Сопутствующие файлы
-------------------

.. dropdown:: 1. :blue:`myalldata.py`

    В данном файле хранятся все данные, которые могут изменяться, а именно:

    1. **url_mapping** - Сайты для определения и переименовывания

    2. **replacements** - Имена сотрудников для соответствий

    3. **bit_replacements** - Словарь для Битрикса

    4. **employee_timezones** - Словарь с часовыми поясами сотрудников

    5. **company_timezone** - Текущий часовой пояс компании

.. dropdown:: 2. :blue:`myallpath.py`

    Данный файл использовался для тестов с путями до файлов. В дальнейшем может использоваться для этого же.

.. dropdown:: Обозначения переменных

    **people** - массив с названиями отделов и сотрудники принадлежащие к ним

    **zvonki** - путь к файлу со звонками из АТС

    **zip_path** - путь к файлу .zip от Стахановца с 4мя отчетами (Программы, Сайты, Лента активности, Табель УРВ)

    **output_excel_path** - путь куда сохранять exel файл

    **mas_sotrudniki** - массив с отделами и именами сотрудников

    **output_xlsx_path** - путь к созданному .xlsx файлу

    **start_path** - путь до распакованного архива Стахановца

    **found_paths** - путь к файлу index.xml содержащим файлы отчетов о Программах и Сайтах

    **output_file_path_prog** - путь к фалу с данными о программах

    **output_file_path_site** - путь к файлу с данными о сайтах

    **program_data** - массив данных о программах

    **site_data** - массив данных о сайтах

    **output_file_excelSite** - путь до нового файла отчета (обработанного)

    **copy_file_path_zv** - путь до копии файла из АТС

    **all_employees** - массив всех сотрудников

    **file_path_bit** - путь до .xlsx фала битрикса
